Controls

each action instantly refreshes the Status panel above
Advance time Move the clock forward by days / months / quarters — crossing a quarter triggers a period transition and usage decay.
Set date Jump the emulator clock to a specific calendar date.
New account Create a SLURM account with an optional node-hour allocation (the budget usage is measured against).
Inject usage Record node-hours of usage for a user in an account at the current time — the account/user are created if missing.
Periodic settings Set an account's fairshare weight and its GrpTRESMins:billing limit (the per-period budget Slurm enforces).
QoS Down: force the account's QoS to slowdown/blocked (throttle or stop new jobs). Restore: return it to normal.
Scenario Run a predefined test scenario (creates accounts, injects usage, advances time, checks QoS) against the live state.
